# UI Audit

Use this skill when the task is to judge UX/UI quality from what is actually rendered in the browser.

## Workflow

1. Capture top, middle, and lower-page screenshots.
2. Evaluate the page as a user first:
   - what stands out
   - what the primary action seems to be
   - what feels confusing
3. Evaluate the interface as a designer second:
   - hierarchy
   - spacing
   - CTA consistency
   - visual rhythm
   - information architecture
4. Finish with a short list of changes in priority order.

## Tone

- direct
- concrete
- focused on outcomes, not taste alone
